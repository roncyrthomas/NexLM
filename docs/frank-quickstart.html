The three models at a glance

Model Tiers active Trainable params Tests What it isolates
Vanilla LoRA only ~3M (LoRA r=16 on Q/K/V/O) Standard finetune control The base model's own quality
Frank v1 + Hormones, Hebbian, HippoRAG, Titans, online-LoRA ~3M LoRA + ~15M Titans + ~3MB Hebbian External-reward agent layer Does our agent layer add value?
Frank v2 + Predictive, Episodic, Habits, Dreamer, Metaplastic ~3M LoRA + ~15M Titans + ~6MB sparse state Self-derived learning signals Do self-derived signals beat reward-only?
The headline experiment is the (Frank v2 − Frank v1) delta. That isolates the contribution of self-learning beyond external reward. The (Frank v1 − Vanilla) delta confirms the agent layer itself helps.

Architecture details, side-by-side

Figure 1. Same frozen base, same LoRA, increasing layer richness. v1 adds five tiers reacting to user reward; v2 adds five more tiers driven by the system's own predictive error.

2.1   Frank v1 tiers (already implemented)

TierFileWhat it doesParam cost
0aagent/hormones.py5 EMAs (joy/frustration/confidence/fatigue/boredom) modulate LR, sampling temp, surprise thresholds, snapshot triggers.~0 (5 floats)
0bagent/hebbian.pyDense 850×850 matrix; three-factor rule (pre×post×reward) builds task↔tool associations.~3 MB
1agent/hipporag.pyOpenIE-extracted knowledge graph; Personalized PageRank retrieval over query-entity seeds.scales with docs
2agent/titans.pyMLP with WK/WV projections; surprise-gated test-time updates; hooks into chosen base-model layers.~15 M
3PEFT LoRA in wrapper.pyRank-16 LoRA on Q/K/V/O; snapshot ring for rollback; sleep training nightly.~3 M

2.2   Frank v2 tiers (newly added)

TierFileWhat it doesParam cost
Pagent/predictive.pyBase model predicts next user input; surprise = −log p(actual) normalized via EMA z-score → sigmoid. Feeds hormones without external reward.0 (reuses base)
Eagent/episodic.pyHash-embedding + scipy KDTree cache of (query, response, reward); KNN recall biases generation toward proven good responses.~100 MB at 10k
Hagent/habits.pyCompiled (intent, tool, shape) triples bypass the generation loop entirely on familiar inputs. Frustration-inhibited.~0 (sparse dict)
Dagent/dreamer.pyWhen fatigue high, sample episodes → replay through agent → LoRA gradient step on the predictive error.0 new params
Magent/metaplastic.pyPer-pair Hebbian learning rate; bumped up on retrospectively-useful updates, down on noisy ones.~3 MB

2.3   The single-turn pipeline (Frank v2 order)

1.  Habit bypass check        (v2) — compiled triple? Return cached response, skip gen.
2.  Episodic recall            (v2) — KNN past similar; positive hits → few-shot hint.
3.  HippoRAG retrieve          (v1) — KG retrieval, top-k chunks.
4.  Tool descriptions          (v1) — catalogue rendering.
5.  Hormone-modulated temp     (v1).
6.  Predictive forecast        (v2) — cache p(next user input) for surprise scoring.
7.  Generate.
8.  Parse + return.

After user reaction (observe_feedback):
  - Predictive surprise from actual user message → feeds hormones.
  - Hebbian update on active features (modulated by metaplastic η[i,j]).
  - Episodic store.
  - Habits observe + decay tick.
  - Dreamer fires if fatigue ≥ threshold.

Installation & setup

Local development (CPU/MPS/8GB GPU)

git clone https://github.com/roncyrthomas/NexLM.git
cd NexLM

# Python 3.10-3.12, venv recommended
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\Activate.ps1

# Torch (pick the wheel for your GPU)
pip install torch                                                          # CPU only
# OR
pip install torch --index-url https://download.pytorch.org/whl/cu124       # consumer GPU
# OR
pip install torch --index-url https://download.pytorch.org/whl/cu130       # RTX 50xx (Blackwell)

pip install -r requirements.in
pip install scipy networkx peft                                            # if not in requirements

# Sanity check
python -m pytest tests/ -v -m "not slow"     # should show 114 passed

Cloud (Linux + A100/A10 + bitsandbytes)

uv pip install torch --index-url https://download.pytorch.org/whl/cu124 --system
uv pip install -r requirements.in --system
uv pip install bitsandbytes --system          # 4-bit quantization for Phi-3-mini

5-minute quickstart code

3.1   Instantiate all three agents

from agent import AgentConfig, NexAgent

# Same base model, three configurations
vanilla  = NexAgent(AgentConfig.vanilla()).cuda()
frank_v1 = NexAgent(AgentConfig.frank_v1()).cuda()
frank_v2 = NexAgent(AgentConfig.frank_v2()).cuda()

# Override the base if you want Phi-3-mini for the paper
vanilla = NexAgent(AgentConfig.vanilla(base="microsoft/Phi-3-mini-4k-instruct")).cuda()

3.2   Train all three with the same data

# From the command line
python train/train_three.py \
    --base smollm2 \
    --variants vanilla,frank_v1,frank_v2 \
    --steps 500 \
    --dataset HuggingFaceH4/ultrachat_200k \
    --batch_size 1 --grad_accum 8 --lr 2e-4 \
    --out_root out/three_way \
    --wandb

3.3   Run the three-way comparison

from evals.runner import compare_three

results = compare_three(
    vanilla=vanilla,
    frank_v1=frank_v1,
    frank_v2=frank_v2,
    benchmarks=["bfcl", "musique", "trace"],
    max_examples=100,
)

# results["delta_v1_vs_vanilla"]["bfcl"]["tool_match"]   # paper claim 1
# results["delta_v2_vs_v1"]["trace"]["avg_accuracy"]     # paper claim 2 (key)
# results["delta_v2_vs_v1"]["musique"]["exact_match"]    # paper claim 3

3.4   A full interaction with feedback

# Set up tools + docs (optional)
frank_v2.tools.register("get_weather", lambda city: f"Sunny in {city}",
                        description="Get current weather",
                        parameters={"type":"object","properties":{"city":{"type":"string"}}})

frank_v2.hipporag.ingest_documents(
    [("doc1", "Tokyo is the capital of Japan.")],
    extractor=lambda text: [{"subject":"Tokyo","predicate":"is","object":"capital of Japan","confidence":0.9}],
)

# Single turn
out = frank_v2.turn("What is the capital of Japan?", intent_id=42)
print(out["response"])
print(out["retrieved_chunks"])

# Feedback loop — drives all the self-learning tiers
frank_v2.observe_feedback(
    reward=1.0,                  # user was happy
    active_features=[42, 7],     # intent + tool ids that co-fired
    last_user_query="What is the capital of Japan?",
    last_response=out["response"],
    next_user_message="Thanks!", # used for predictive surprise
    intent_id=42, tool_id=7, shape_id=2,
)

4.3   Training procedure step by step

  1. SFT on a shared corpus. All three agents see the same data, same step count, same hyperparams. This isolates the agent layer from data variance.
  2. (Frank v1, v2 only) prime the agent layer with simulated feedback. Stream a few hundred annotated examples through observe_feedback() so the Hebbian H and (v2) episodic memory have some content.
  3. (Frank v2 only) let it dream once. Force dreamer.dream(agent) to run a single consolidation pass before evaluation.
  4. Run the three-way compare on the held-out benchmark splits. 3 seeds × 200 examples each.
  5. Tabulate the deltas. Per benchmark, per metric.

Custom probes the harness supports

  • Latency comparison — measure first-token latency under all three models. v2 should be 30–50% faster on familiar inputs (habits bypass).
  • Forgetting probe — after 1000 turns of TRACE, retest task 1 accuracy. v2 should retain it; v1 should drift.
  • Predictive surprise distribution — log surprise per turn over time; sharpens for v2 as it adapts.

Troubleshooting

"OutOfMemoryError" loading Phi-3-mini locally

Phi-3-mini is 3.8B params. On a 5060-class GPU, switch to SmolLM2-1.7B for local dev:

cfg = AgentConfig.frank_v2(base="HuggingFaceTB/SmolLM2-1.7B-Instruct")

Or on cloud with bitsandbytes: cfg.quantization = "4bit".

"ModuleNotFoundError: No module named 'mamba_ssm'"

You don't need it for any of v1 or v2. The mamba/diff-attn backbone (M1a/M1b) was the original architecture path; the pivoted agent layer uses any pretrained base. If something tries to import it, you're on M1a code by accident.

HF Hub rate-limiting

Set an HF_TOKEN: export HF_TOKEN=hf_xxx. Without it, downloads are slower and may fail for large datasets.

"undefined symbol" from causal_conv1d / mamba_ssm on cloud

ABI mismatch — torch was upgraded after those packages compiled. Uninstall both: pip uninstall -y mamba-ssm causal-conv1d. The agent layer doesn't need them.

Wandb keeps asking for API key

wandb login once; key cached at ~/.netrc.

compare_three returns empty deltas

The benchmark fell back to inline 3–4 examples. Check results["vanilla"]["bfcl"]["source"] — if it says inline_fallback, HF dataset load failed. Try setting HF_TOKEN or pre-downloading via datasets-cli.
